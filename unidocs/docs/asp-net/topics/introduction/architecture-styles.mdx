# Архитектурни стилове - MVC, API и Razor Pages 

## MVC 

**MVC (Model, View, Controller)** е шаблон за софтуерен дизайн, използван за разделяне на потребителския интерфейс (изглед - View), данните (модел - Model) и логиката на приложението (контролер - Controller). ASP.NET предоставя мощен, базиран на шаблони начин за изграждане на динамични уеб приложения, използвайки MVC шаблона, който позволява ясно разделяне на отговорностите.

Чрез MVC шаблона, заявките се насочват към контролер, който е отговорен за работата с модела за извършване на действия и/или извличане на данни. Контролерът избира изгледа за показване и му предоставя модела. Изгледът извежда (render) крайната страница въз основа на данните в модела.

### Models 

Моделът в MVC приложение представлява състоянието на приложението и всяка бизнес логика или операции, които трябва да бъдат изпълнени от него. Бизнес логиката трябва да бъде капсулирана в модела, заедно с всяка логика за имплементация за запазване на състоянието на приложението. 

Силно типизираните изгледи обикновено използват типове ViewModel, предназначени да съдържат данните, които да се показват в този изглед. Контролерът създава и попълва тези ViewModel екземпляри от модела.

Използват се класове, които се свързват с базата данни. Могат да се дефинират правила за валидиране, използвайки C# атрибути, които се прилагат на клиента и сървъра.

ASP.NET поддържа много database engines: SQLite, SQL Server, MySQL, PostgreSQL, DB2 и други, както и нерелационни хранилища като MongoDB, Redis и Azure Cosmos DB.

##### Пример

```csharp
public class Student
{
    public string StudentId { get; set; }

    [Required]
    [MinLength(2)]
    public string Name { get; set; }

    [Phone]
    public string PhoneNumber { get; set; }

    [EmailAddress]
    public string Email { get; set; }
}
```

### Views

Изгледите са отговорни за представяне на съдържанието чрез потребителски интерфейс. Използват се Razer страници, които вграждат .NET код директно в HTML кода. Логиката в изгледите трябва да е минимална и всяка логика в тях трябва да бъде свързана с извеждане на съдържанието. 

При нужда от използване на повече логика с цел извеждане на информация от сложен модел, се използва View Component, ViewModel или View Template, които целят опростяване на изгледа.

### Controllers

Контролерите са компонентите, които обработват взаимодействията с потребителя, работят с моделите и накрая избират изгледа, който да бъде показан. В MVC приложенията, изгледите единствено показват информацията; контролерите обработват и реагират на потребителския вход и взаимодействие.

Контролерът е началната входна точка и е отговорен това с кои модели да работи и кой изглед да бъде показан. Оттам идва и името му - контролира как приложението отговаря на дадена заявка.

Заявките се насочват към контролерите, които съдържат actions, имплементирани като нормални C# методи. Данните от пътя на заявката и нейното тяло се обвързват автоматично с параметрите на метода.

:::info
Контролерите не бива да бъде натоварени с твърде много отговорности. За да избегнем сложна логика в контролерите, пренасяме бизнес логиката в **Services**.
:::

#### Пример

```csharp
public class StudentController : Controller
{
    private readonly ApplicationDbContext _context;

    public StudentController(ApplicationDbContext context)
    {
        _context = context;
    }

    // GET: /student
    public async Task Index()
    {
        return View(await _context.Student.ToListAsync());
    }

    // GET: /student/details/5
    public async Task Details(int id)
    {
        var student = await _context.Student.Find(id);

        if (student == null)
        {
            return NotFound();
        }

        return View(student);
    }
}
```

## API

**API (Application Programming Interface)** е интерфейс, чрез който едно приложение предоставя данни и функционалност на други приложения - най‑често чрез HTTP. За разлика от MVC и Razor Pages, които връщат HTML към браузъра, API връща данни, обикновено във формат JSON.

API е бекенд без изгледи, който приема заявки и връща структурирани отговори. Той се използва, когато клиентът не е задължително браузър - може да е мобилно приложение, JavaScript front-end (React, Angular, Vue), друго back-end приложение или външна система.

### Основни Характеристики

- Работи чрез HTTP (GET, POST, PUT, DELETE).
- Връща JSON, а не HTML.
- Използва контролери, но без изгледи.
- Подходящо е за SPA приложения, мобилни приложения, интеграции.
- Поддържа Model Binding и Model Validation, както и MVC.
- Често използва DTOs вместо ViewModels.
- Има поддръжка за Authentication и Authorization чрез токени (JWT).

#### Пример

```csharp
[ApiController]
[Route("api/[controller]")]
public class BooksController : ControllerBase
{
    [HttpGet]
    public IActionResult GetAll()
    {
        return Ok(new { Title = "Clean Code", Author = "Robert C. Martin" });
    }
}
```

### Кога използваме API вместо MVC

**Използваме API, когато:**
- Клиентът не е браузър.
- Имаме нужда от данни, а не HTML.
- Правим SPA (React, Angular, Vue).
- Правим мобилно приложение.
- Правим интеграция между системи.

**Използваме MVC, когато:**
- Връщаме HTML.
- Имаме изгледи.
- Правим уеб приложение.

:::note
API е back-end, който говори с други приложения, а не с браузъра.

MVC е back-end, който връща HTML към браузъра.
:::

## Razor Pages

Razor Pages е архитектурен модел в ASP.NET Core, който позволява изграждане на уеб приложения чрез страници, вместо чрез контролери и изгледи, както е при MVC. Той е по‑опростен, по‑структуриран и по‑подходящ за приложения, които са ориентирани около страници, а не около контролери.

Всяка страница има собствен файл за изглед (.cshtml) и собствен файл за логика (.cshtml.cs).  
Това прави структурата по‑чиста и по‑лесна за разбиране, особено за начинаещи.

### Как изглежда Razor Page

**Index.cshtml**

```html
@page
<h1>Hello from Razor Pages</h1>
```

**Index.cshtml.cs**

```csharp
public class IndexModel : PageModel
{
    public void OnGet()
    {
    }
}
```

- @page казва, че това е Razor Page.
- `OnGet()`, `OnPost()` и др. са еквивалент на action методи.

## Razor Views vs Razor Pages

### Razor Views

- Част от ASP.NET Core MVC.
- Използват се заедно с контролери.
- Контролерът връща изглед чрез `return View()`.
- Файловете са в `/Views/ControllerName/`.
- Използват се за извеждане на HTML.

### Razor Pages

- Не използва контролери.
- Всяка страница има собствена логика.
- Файловете са в `/Pages/`.
- Съдържа 2 файла:
    - Page.cshtml -> HTML.
    - Page.cshtml.cs -> логика (PageModel).
- Методите са OnGet(), OnPost(), OnGetAsync().

|**Концепция**|**Razor Views**|**Razor Pages**|
|-------------|---------------|---------------|
|_Архитектура_|MVC|Алтернативен модел|
|_Контролери_|Да|Не|
|_Логика_|В контролера|В PageModel|
|_Файлове_|/Views/ControllerName/..|/Pages/..|
|_Подход_|Controller -> View|Page -> PageModel|
|_Използване_|Класическо уеб приложение|опростено приложение с CRUD операции|
