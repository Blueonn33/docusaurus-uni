# Приложен слой (Application Layer)

Приложният слой представлява централната част на бизнес логиката в многослойната архитектура. Той дефинира какво прави приложението, без да се интересува как точно се изпълнява.
Тук се намират услугите (Services), които координират операции, използват Repositories, прилагат бизнес правила и управляват потока от данни между Уеб слоя и Инфраструктурния слоя.

Приложният слой е напълно независим от инфраструктурата и базата данни. Той работи само чрез абстракции (интерфейси), което позволява лесно тестване, подмяна на имплементации и чиста архитектура.

---

## Основни отговорности

- Съдържа бизнес логика, която не принадлежи на Домейн слоя.
- Управлява операции, които включват множество домейн обекти.
- Работи чрез интерфейси, а не чрез конкретни имплементации.
- Използва Repositories за достъп до данни.
- Извършва валидации, специфични за бизнес процесите.
- Използва DTOs за вход и изход.
- Не съдържа UI логика.
- Не съдържа логика за достъп до база данни.

---

## Елементи на слоя

- **Application Services** - Основните класове, които изпълняват бизнес операции.
- **Interfaces** - Дефинират какво може да прави дадена услуга.
- **DTOs** - Модели за вход и изход.
- **Validators** - Бизнес валидации.
- **Mapping** - Преобразуване между DTOs и домейн модели.

---

## Роля в многослойната архитектура

Приложният слой е междинният (средният) слой, който свързва Уеб слоя с Инфраструктурния слой.

**Потокът е следният:**

1. Уеб слоят приема заявка.
2. Извиква метод от Приложния слой.
3. Приложният слой:
    - Валидира входните данни.
    - Изпълнява бизнес логика.
    - Използва Repositories чрез интерфейси.
4. Връща резултат към Уеб слоя

По този начин Приложният слой гарантира, че бизнес логиката е централизирана, може да бъде лесно тествана и е независима от инфраструктурата.

#### Пример

```csharp
public class BookService : IBookService
{
    private readonly IBookRepository _repository;

    public BookService(IBookRepository repository)
    {
        _repository = repository;
    }

    public IEnumerable<BookDto> GetAll()
    {
        var books = _repository.GetAll();
        return books.Select(b => new BookDto
        {
            Id = b.Id,
            Title = b.Title,
            Author = b.Author
        });
    }
}
```
